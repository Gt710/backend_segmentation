# Розділ. Програмна реалізація системи підтримки прийняття рішень та навчання моделей штучного інтелекту

## 1. Загальна архітектура програмного комплексу

Розробка медичної інформаційної системи для сегментації пухлин головного мозку вимагає поєднання складних математичних обчислень (машинне навчання) зі зручним та швидким інтерфейсом користувача, який відповідає вимогам радіологічних відділень. Враховуючи ці специфічні вимоги, для програмної реалізації було обрано клієнт-серверну архітектуру на локальному рівні. 

Такий підхід дозволив чітко розділити систему на два незалежні модулі. Перший модуль (бекенд) реалізований мовою Python з використанням фреймворку PyTorch та відповідає виключно за обробку МРТ-знімків і роботу нейромережевої моделі. Другий модуль (фронтенд) розробляється з використанням сучасної технології Kotlin Jetpack Compose для створення кросплатформного десктопного застосунку. Взаємодія між цими модулями відбувається через REST API (на базі FastAPI), що забезпечує високу масштабованість та унеможливлює конфлікти між залежностями віртуальної машини Java (JVM) та бібліотеками для роботи з тензорними обчисленнями.

## 2. Підсистема збору, індексації та попередньої обробки медичних даних

Основою для навчання моделі штучного інтелекту виступає набір даних BraTS (Brain Tumor Segmentation). Враховуючи великий обсяг повного датасету, програмно реалізовано скрипт автоматизованого завантаження підмножини даних (100 пацієнтів) за допомогою бібліотеки `huggingface_hub`. Скрипт `download_subset.py` налаштовано на отримання репрезентативної вибірки з категорій GLI та MEN-RT, забезпечуючи наявність чотирьох модальностей МРТ (T1, T1c, T2, FLAIR) та відповідної маски сегментації для кожного випадку.

Для оптимізації доступу до даних під час навчання розроблено модуль індексації (`create_index.py`). Він рекурсивно обходить завантажені директорії, зіставляє файли різних модальностей для кожного пацієнта та формує єдиний реєстр у форматі CSV (`dataset_index.csv`). Такий підхід усуває необхідність постійного сканування файлової системи під час тренувального циклу, що значно прискорює процес генерації батчів (пакетів даних).

Медичні зображення у форматі NIfTI містять значну кількість фонового простору (пустот навколо мозку), який не несе корисної інформації, але споживає обчислювальні ресурси. Для вирішення цієї проблеми у модулі `preprocessing.py` реалізовано алгоритм автоматичного визначення обмежувального контейнера (Bounding Box). Алгоритм аналізує пікселі, відмінні від нуля, знаходить мінімальні та максимальні координати по всіх трьох осях, після чого виконує просторове обрізання 3D-об'єму.

Наступним критичним етапом попередньої обробки є нормалізація інтенсивності вокселів. Оскільки МРТ-сканери різних виробників генерують знімки з різним діапазоном значень яскравості, подача сирих даних у нейромережу призвела б до нестабільності навчання. Було імплементовано метод Z-score нормалізації, який обчислює середнє значення та стандартне відхилення виключно для тканин мозку (ігноруючи фон), після чого приводить дані до стандартизованого розподілу.

Для подачі даних у модель реалізовано клас `BraTSDataset`, успадкований від `torch.utils.data.Dataset`. Враховуючи обмеження відеопам'яті (зокрема, використання графічного прискорювача NVIDIA GeForce RTX 4060 Laptop), подача цілих 3D-сканів у пам'ять є неможливою. Тому завантажувач даних реалізує алгоритм випадкового вилучення 3D-патчів фіксованого розміру (64x64x64 вокселі) з чотирьох модальностей. Це не лише вирішує проблему пам'яті, але й виконує роль аугментації даних, оскільки в кожну епоху модель "бачить" трохи зміщені фрагменти пухлин.

## 3. Проектування та програмна реалізація архітектури 3D U-Net

Для виконання задачі воксельної сегментації було спроектовано нейронну мережу класу 3D U-Net (`src/models/unet3d.py`). Архітектура побудована мовою Python з використанням базових примітивів бібліотеки PyTorch (`torch.nn`). 

Мережа складається з двох основних частин: симетричного енкодера (шляху стиснення) та декодера (шляху відновлення). Для створення глибокої структури розроблено базовий блок `ConvBlock3D`, який інкапсулює дві послідовні тривимірні згортки (`Conv3d`), нормалізацію батчу (`BatchNorm3d`) та функцію активації `ReLU`. Використання нормалізації батчу дозволило прискорити збіжність моделі та зробити процес навчання менш чутливим до початкової ініціалізації ваг.

Вхідний шар мережі приймає тензори розмірністю `[Batch, 4, 64, 64, 64]`, де 4 — це кількість МРТ-модальностей. Енкодер поступово зменшує просторову роздільну здатність за допомогою операцій `MaxPool3d`, одночасно збільшуючи кількість каналів ознак (features) від 32 до 128. Це дозволяє моделі вивчати складні просторові та семантичні патерни злоякісних новоутворень.

Декодер мережі використовує операції транспонованої згортки (`ConvTranspose3d`) для відновлення просторової розмірності. Критично важливим елементом реалізації стали так звані "skip connections" (прокидання зв'язків) — конкатенація карт ознак з відповідного рівня енкодера до поточного рівня декодера. Цей архітектурний прийом дозволяє поєднати високорівневий семантичний контекст (розуміння "що це") з низькорівневою просторовою інформацією (розуміння "де точно знаходяться межі").

Фінальний шар мережі являє собою згортку `Conv3d` з розміром ядра 1x1x1, яка зводить багатовимірний тензор ознак до єдиного каналу. До виходу застосовується функція активації `Sigmoid`, яка перетворює значення вокселів у діапазон [0, 1], що інтерпретується як ймовірність належності кожного вокселя до класу пухлини.

## 4. Процес оптимізації та навчання моделі

Процес навчання (тренувальний цикл) формалізовано у скрипті `src/train.py`. Однією з головних проблем сегментації медичних зображень є сильний дисбаланс класів: об'єм пухлини зазвичай становить менше 5% від загального об'єму мозку на знімку. Стандартна функція втрат (наприклад, бінарна крос-ентропія) в таких умовах змушує модель оптимізувати прогноз у бік "фону", ігноруючи пухлину. 

Для вирішення цієї проблеми програмно реалізовано власну функцію втрат `DiceLoss`, яка базується на коефіцієнті подібності Соренсена-Дайса (Sørensen-Dice coefficient). Ця функція вимірює просторове перекриття між прогнозованою маскою та істинною розміткою (ground truth), ігноруючи правильно класифікований фон. До формули додано параметр згладжування (`smooth`), який запобігає діленню на нуль у випадках, коли на патчі відсутні фрагменти пухлини.

Оптимізація ваг нейронної мережі здійснюється за допомогою алгоритму Adam (Adaptive Moment Estimation) з базовою швидкістю навчання 0.001. Цей алгоритм динамічно адаптує швидкість навчання для кожного параметра, що особливо ефективно для складних архітектур зі значною кількістю ваг.

Тренувальний цикл розділений на етапи навчання та валідації. Для забезпечення об'єктивної оцінки датасет розділено у пропорції 80/20 (тренування/валідація) за допомогою `random_split`. Під час ітерацій по тренувальним батчам модель обчислює градієнти (`loss.backward()`) та оновлює ваги (`optimizer.step()`). Після кожної епохи автоматично запускається цикл валідації без обчислення градієнтів (`torch.no_grad()`), що дозволяє моніторити здатність моделі до узагальнення (генералізації) та вчасно виявляти ознаки перенавчання. Програмна реалізація автоматично визначає доступність графічного прискорювача та переносить усі обчислення на GPU (`cuda`), що підтверджено успішним виконанням тренувального циклу. Після завершення навчання ваги моделі зберігаються у бінарному форматі `.pth`.

## 5. Розробка інтеграційного середовища та бекенду

Для забезпечення доступу до натренованої моделі з боку графічного інтерфейсу створено модуль `api.py`. Його реалізовано за допомогою високопродуктивного асинхронного веб-фреймворку FastAPI. При запуску сервер автоматично інстанціює клас `UNet3D`, завантажує збережені ваги моделі та переводить мережу в режим інференсу (`model.eval()`). 

Створено кінцеву точку (endpoint) `POST /predict`, яка налаштована на прийом багаточастинних форм (multipart/form-data) з чотирма NIfTI-файлами. Інтеграція через FastAPI дозволила реалізувати автоматичну генерацію документації Swagger UI та створити надійний міст між середовищами розробки.

Десктопний графічний інтерфейс користувача розробляється з використанням фреймворку Kotlin Jetpack Compose для JVM. Програмний каркас UI містить адаптивне компонування (`Column`, `Row`, `Box`), яке дозволяє масштабувати вікно програми під різні дисплеї. Розроблено базову структуру, що включає зону для відображення мультипланарної реконструкції знімків та панель керування для завантаження файлів пацієнта. Взаємодія з бекендом проектується через HTTP-клієнт Ktor, що забезпечить асинхронне відправлення знімків на Python-сервер та отримання розрахованої 3D-маски без блокування основного потоку інтерфейсу (UI thread). 

Комплексний підхід до програмної реалізації, застосований у системі, гарантує швидкість обробки медичних зображень, точність роботи моделі штучного інтелекту та можливість легкої модифікації як інтерфейсної частини, так і алгоритмів машинного навчання у майбутньому.
