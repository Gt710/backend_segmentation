Я реалізував логіку прогнозування характеру пухлини (доброякісна/злоякісна) на основі історії аналізів, як ви описали у файлі!

Ось що було зроблено:

1.  **База даних (`src/database.py`)**:
    *   До моделі `Scan` додано нове поле `tumor_nature`, де зберігатиметься результат оцінки (Доброякісна, Злоякісна або Недостатньо даних).
2.  **Бекенд-логіка (`src/api.py`)**:
    *   Тепер при кожному аналізі (ендпоінт `/analyze`) сервер шукає **попередні успішні аналізи** цього ж пацієнта.
    *   Якщо це **перший аналіз** — система ставить статус *"Недостатньо даних"*.
    *   Якщо є **попередній аналіз** — система порівнює новий об'єм із попереднім:
        *   Якщо об'єм **збільшився** — висновок: *"Злоякісна (прогресування)"*.
        *   Якщо об'єм **зменшився або не змінився** — висновок: *"Доброякісна або регресія"*.
3.  **Звіти та відповіді**:
    *   Цей висновок додано у відповідь API (поле `"tumor_nature"`).
    *   Також пункт **"Характер (динаміка)"** тепер відображається у згенерованому PDF-звіті.

> [!IMPORTANT]
> Оскільки ми змінили структуру бази даних (додали нову колонку), SQLAlchemy може видати помилку при спробі запису, якщо файл бази вже існує. Якщо виникне помилка, просто **видаліть файл `data/segmentation.db`**, щоб сервер створив його заново з новою структурою.

Сервер автоматично перезавантажився. Ви можете протестувати це, зробивши два аналізи для одного й того ж пацієнта (наприклад, з різними номерами знімків або просто повторно) через `test_api_db.py`!
